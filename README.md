# enterprise-bd-os

An autonomous B2B business development operating system.

Finds your ICP companies → predicts buying signals → reaches out via LinkedIn + email → generates GTM content → monitors itself → self-optimizes with an AI super-agent.

## Quick Start

```bash
git clone https://github.com/YOUR_USERNAME/enterprise-bd-os
cd enterprise-bd-os
pnpm install
pnpm run setup    # 3-minute interactive setup wizard
pnpm db:migrate --name init
pnpm db:generate
pnpm start
```

## What It Does

| Engine | What it does | Schedule |
|--------|-------------|----------|
| **Data Engine** | Ingests articles + signals from 20+ sources (HackerNews, NewsAPI, RSS) | Every 4h |
| **Prediction Engine** | Scores companies by expansion likelihood using configurable signal types | 5 AM daily |
| **BD Engine** | Discovers leads (Apollo), enriches contacts, sends LinkedIn + email outreach | 9–10 AM daily |
| **GTM Engine** | Generates SEO blog posts + LinkedIn content from prediction data | 2 PM daily |
| **Monitoring Engine** | Health checks across all engines, alerts on degraded state | 6 PM daily |
| **Super-Agent** | LangGraph AI orchestrator that self-optimizes the entire system | 4× daily |

## Architecture

```
Data Engine → Prediction Engine → BD Engine → GTM Engine → Monitoring → Super-Agent
     ↑                                                                       ↓
     └──────────────────── SQLite (Prisma) ◄──────────────────────────────────┘
```

All state persists in a local SQLite database. The super-agent reads system state after each engine run and uses Claude (or any LLM) to optimize connection limits, message templates, scoring thresholds, and content angles.

## Configuration

All configuration lives in `config/`. Generated by `pnpm run setup`:

| File | What it configures |
|------|--------------------|
| `config/icp.yaml` | Your ideal customer profile (industries, company size, geographies, signals) |
| `config/predictions.yaml` | Custom prediction types, signal weights, scoring thresholds |
| `config/outreach.yaml` | LinkedIn + email templates, daily limits |
| `config/sources.yaml` | Which data sources to enable |
| `config/model.yaml` | LLM provider and model |

## LLM Provider

Swap LLM providers by editing `config/model.yaml`:

```yaml
# Anthropic (default)
provider: anthropic
model: claude-sonnet-4-6
api_key_env: ANTHROPIC_API_KEY

# OpenAI
provider: openai
model: gpt-4o
api_key_env: OPENAI_API_KEY

# Groq (fast inference)
provider: groq
model: llama-3.1-70b-versatile
api_key_env: GROQ_API_KEY

# Ollama (local, free)
provider: ollama
model: llama3.2
api_key_env: OLLAMA_API_KEY
base_url: http://localhost:11434
```

## Required API Keys

| Key | Purpose | Required? |
|-----|---------|-----------|
| Anthropic / OpenAI / Groq key | LLM reasoning (super-agent + GTM) | Yes (one of these) |
| `APOLLO_API_KEY` | Lead discovery + contact enrichment | Yes |
| `NEWSAPI_KEY` | News signal ingestion | Recommended |
| SMTP credentials | Email outreach | For email outreach |
| `SALESQL_API_KEY` | Contact enrichment (fallback) | Optional |

## Customizing Prediction Types

Edit `config/predictions.yaml` to define your own signals:

```yaml
prediction_types:
  - name: VENDOR_EVALUATION
    description: "Company actively evaluating new software vendors"
    signals: ["rfp_signal", "job_posting_vendor", "budget_cycle"]
    weight: 0.7
  - name: GROWTH_MODE
    description: "Company in rapid growth phase"
    signals: ["hiring_surge", "funding_round", "new_office"]
    weight: 0.3
```
